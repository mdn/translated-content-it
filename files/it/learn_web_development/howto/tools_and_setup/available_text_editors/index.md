## Riassunto

Un sito web è costituito principalmente da file di testo, quindi per un'esperienza di sviluppo divertente e piacevole è importante scegliere con saggezza il proprio editor di testo.

L'enorme numero di scelte può essere un po' travolgente, dal momento che un editor di testo è un elemento di base nell'informatica (sì, lo sviluppo web è informatica). Idealmente, si dovrebbe provare il maggior numero possibile di editor e capire quale si adatta meglio al proprio flusso di lavoro. Ma le daremo alcuni suggerimenti per iniziare.

Ecco le principali domande da considerare:

- Con quale sistema operativo voglio lavorare?
- Che tipo di tecnologie voglio manipolare?
- Quali funzionalità base mi aspetto dal mio editor di testo?
- Voglio aggiungere funzionalità extra al mio editor di testo?
- Ho bisogno di supporto/aiuto mentre uso il mio editor di testo?
- L'aspetto del mio editor di testo è importante per me?

Noti che non abbiamo menzionato il prezzo. Ovviamente è importante anche quello, ma il costo di un prodotto ha poco a che fare con la qualità o la capacità. C'è una buona possibilità di trovare un editor di testo adatto gratuitamente.

## Apprendimento attivo

In questa sezione di apprendimento attivo, desideriamo che lei provi a utilizzare e/o installare un editor di testo a sua scelta. Il suo computer potrebbe già avere uno degli editor suggeriti sopra (ad esempio, Gedit se utilizza il desktop GNOME, Kate se utilizza KDE, ecc.), altrimenti dovrebbe provare a installare uno o più editor di testo di sua scelta.

Provi ad esplorare le impostazioni del suo editor e legga il manuale o la documentazione per vedere quali sono le sue funzionalità. In particolare (se possibile nel suo editor), cerchi di:

- Cambiare le impostazioni e i colori dell'evidenziazione della sintassi
- Giocare con la larghezza dell'[indentazione](<https://en.wikipedia.org/wiki/Indentation_(typesetting)#Indentation_in_programming>), impostando un'opzione appropriata per le sue esigenze
- Verificare le impostazioni di salvataggio automatico e salvataggio sessione
- Configurare eventuali [plugin](<https://en.wikipedia.org/wiki/Plug-in_(computing)>) disponibili e indagare su come ottenere nuovi plugin
- Cambiare schemi di colore
- Regolare le impostazioni di visualizzazione e vedere come può cambiare il layout delle visualizzazioni
- Verificare quali lingue di programmazione/tecnologie supporta il suo editor

Mentre sta imparando, le impostazioni predefinite della maggior parte degli editor di testo dovrebbero andare bene da utilizzare, ma è importante diventare familiari con gli strumenti scelti, in modo da poter selezionare il migliore per il suo uso. Imparerà di più sulla personalizzazione dei suoi editor e strumenti man mano che guadagna esperienza e, più importante, imparerà quali caratteristiche sono più utili ai suoi scopi.

## Approfondimenti

### Criteri di scelta

Quindi, più in dettaglio, cosa dovrebbe considerare quando sceglie un editor di testo?

#### Con quale sistema operativo voglio lavorare?

Ovviamente è una sua scelta. Tuttavia, alcuni editor sono disponibili solo per determinati sistemi operativi, quindi se le piace passare da uno all'altro, questo potrebbe restringere le possibilità. Qualsiasi editor di testo _può_ svolgere il lavoro, se funziona sul suo sistema, ma un editor multipiattaforma facilita la migrazione da un sistema operativo all'altro.

Quindi innanzitutto scopra quale sistema operativo sta utilizzando e poi controlli se un dato editor supporta il suo sistema operativo. La maggior parte degli editor specifica sul proprio sito web se supportano Windows o Mac, anche se alcuni supportano solo determinate versioni. Se sta eseguendo Ubuntu, la sua migliore opzione è cercare all'interno dell'Ubuntu Software Center. In generale, ovviamente, il mondo Linux/UNIX è un luogo piuttosto diversificato in cui diverse distribuzioni funzionano con diversi sistemi di packaging incompatibili. Ciò significa che, se si è innamorati di un editor di testo poco conosciuto, potrebbe doverlo compilare da sorgente da sola (non per i deboli di cuore).

#### Che tipo di tecnologie voglio manipolare?

In generale, qualsiasi editor di testo può aprire qualsiasi file di testo. Questo funziona bene per scrivere note a sé stesso, ma quando si sviluppano siti web e si scrive in {{Glossary("HTML", "HTML")}}, {{Glossary("CSS", "CSS")}} e {{Glossary("JavaScript", "JavaScript")}}, si possono produrre file piuttosto grandi e complessi. È possibile semplificarsi la vita scegliendo un editor di testo che comprenda le tecnologie con cui si lavora. Molti editor di testo la aiuteranno con funzionalità come:

- **[Evidenziazione della sintassi](https://en.wikipedia.org/wiki/Syntax_highlighting).** Rendere il suo file più leggibile codificando a colori le parole chiave in base alla tecnologia in uso.
- **[Completamento del codice](https://en.wikipedia.org/wiki/Autocomplete#In_source_code_editors).** Risparmiarle tempo completando automaticamente le strutture ricorrenti (ad esempio, chiudendo automaticamente i tag HTML o suggerendo valori validi per una determinata proprietà CSS).
- **[Code snippets](https://en.wikipedia.org/wiki/Snippet_(programming)).** Come ha visto quando ha iniziato un nuovo documento HTML, molte tecnologie utilizzano più volte la stessa struttura di documento. Eviti il problema di riscrivere tutto questo usando un code snippet per riempire automaticamente il suo documento.

La maggior parte degli editor di testo supporta ora l'evidenziazione della sintassi, ma non necessariamente le altre due funzionalità. Si assicuri in particolare che il suo editor di testo supporti l'evidenziazione per {{Glossary("HTML", "HTML")}}, {{Glossary("CSS", "CSS")}}, e {{Glossary("JavaScript", "JavaScript")}}.

#### Quali funzionalità base mi aspetto dal mio editor di testo?

Dipende dalle sue esigenze e dai suoi piani. Queste funzionalità sono spesso utili:

- Cerca e sostituisci, in uno o più documenti, basata su {{Glossary("Regular_Expression", "espressioni regolari")}} o altri schemi necessari
- Saltare rapidamente a una determinata linea
- Visualizzare due parti di un documento grande separatamente
- Visualizzare l'HTML come apparirà nel browser
- Selezionare il testo in più punti contemporaneamente
- Visualizzare i file e le directory del suo progetto
- Formattare automaticamente il suo codice con un beautifier di codice
- Controllo ortografico
- Indentazione automatica del codice basata sulle impostazioni di indentazione

#### Voglio aggiungere funzionalità extra al mio editor di testo?

Un editor estensibile viene fornito con meno funzionalità integrate, ma può essere esteso in base alle sue esigenze.

Se non è sicuro di quali funzionalità desidera, o il suo editor preferito manca di tali funzionalità di default, cerchi un editor estensibile. I migliori editor offrono molti plugin e idealmente un modo per cercare e installarne di nuovi automaticamente.

Se le piacciono _molte_ funzionalità e il suo editor va lento a causa di tutti i suoi plugin, provi a utilizzare un [IDE](https://en.wikipedia.org/wiki/Integrated_development_environment) (ambiente di sviluppo integrato). Un IDE fornisce molti strumenti in un'unica interfaccia ed è un po' intimidatorio per i principianti, ma sempre un'opzione se il suo editor di testo sembra troppo limitato. Ecco alcuni popolari IDE:

- [Aptana Studio](https://www.axway.com/en/aptana)
- [Eclipse](https://www.eclipse.org/)
- [NetBeans IDE](https://netbeans.apache.org/)
- [Visual Studio](https://visualstudio.microsoft.com/)
- [WebStorm](https://www.jetbrains.com/webstorm/)

#### Ho bisogno di supporto/aiuto mentre uso il mio editor di testo?

È sempre bene sapere se si può ottenere aiuto o meno quando si utilizza un software. Per gli editor di testo, controlli due tipi diversi di supporto:

1. Contenuto orientato all'utente (FAQ, manuale, aiuto online)
2. Discussione con sviluppatori e altri utenti (forum, email, IRC)

Utilizzi la documentazione scritta quando sta imparando a usare l'editor. Mettiti in contatto con altri utenti se sta risolvendo problemi durante l'installazione o l'utilizzo dell'editor.

#### L'aspetto del mio editor di testo è importante per me?

Beh, è una questione di gusto, ma alcune persone amano personalizzare ogni aspetto dell'interfaccia utente, dai colori alle posizioni dei pulsanti. Gli editor variano ampiamente in flessibilità, quindi controlli in anticipo. Non è difficile trovare un editor di testo che possa cambiare lo schema dei colori, ma se vuole una personalizzazione pesante potrebbe essere meglio optare per un IDE.

### Installazione e configurazione

L'installazione di un editor di testo è solitamente piuttosto semplice. Il metodo varia in base alla piattaforma, ma non dovrebbe essere troppo difficile:

- **Windows.** Gli sviluppatori le forniranno un file `.exe` o `.msi`. A volte il software viene fornito in un archivio compresso come `.zip`, `.7z` o `.rar`, e in quel caso avrà bisogno di installare un programma aggiuntivo per estrarre il contenuto dall'archivio. Windows supporta `.zip` per default.
- **Mac.** Sul sito web dell'editor può scaricare un file `.dmg`. Alcuni editor di testo può trovarli direttamente nell'Apple Store per rendere ancora più semplice l'installazione.
- **Linux.** Nelle distribuzioni più popolari può iniziare con il gestore di pacchetti grafico (Ubuntu Software Center, mintInstall, GNOME Software, ecc.). Può spesso trovare un file `.deb` o `.rpm` per il software preconfezionato, ma nella maggior parte dei casi dovrà utilizzare il server del repository della sua distribuzione o, nel peggiore dei casi, compilare il suo editor da sorgente. Si prenda il tempo di controllare attentamente le istruzioni di installazione sul sito web dell'editor di testo.

Quando installa un nuovo editor di testo, il suo sistema operativo probabilmente continuerà ad aprire i file di testo con l'editor predefinito fino a quando non cambierà l_‘[associazione dei file](https://en.wikipedia.org/wiki/File_association)_._ Queste istruzioni la aiuteranno a specificare che il suo sistema operativo dovrebbe aprire i file con l'editor da lei preferito quando ci fa doppio clic:

- [Windows](https://support.microsoft.com/en-us/windows)

- [macOS](https://support.apple.com/guide/mac-help/choose-an-app-to-open-a-file-on-mac-mh35597/mac)

- Linux

  - [Ubuntu Unity](https://askubuntu.com/questions/289337/how-can-i-change-file-association-globally)
  - [GNOME](https://help.gnome.org/users/gnome-help/stable/files-open.html.en)
  - [KDE](https://userbase.kde.org/System_Settings/File_Associations)

## Prossimi passi

Ora che ha un buon editor di testo, potrebbe prendersi del tempo per finalizzare [il suo ambiente di lavoro di base](/it/docs/Learn_web_development/Howto/Tools_and_setup/set_up_a_local_testing_server), oppure, se vuole divertirsi subito, scrivere [la sua prima pagina web](/it/docs/Learn_web_development/Getting_started/Your_first_website).
